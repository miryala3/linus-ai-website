Installation

Installation Instructions

Get up and running in under 60 seconds on any platform.

macOS — zsh
$ cd ~/Downloads
$ chmod +x linus-ai-macos-arm64
$ ./linus-ai-macos-arm64 --version
LINUS-AI v1.4.0 (darwin-arm64)
 
# Move to PATH for global access:
$ sudo mv linus-ai-macos-arm64 /usr/local/bin/linus-ai
$ linus-ai --chat "Hello, Linus!"
Gatekeeper Notice: On first launch, macOS may show a security warning. Right-click the binary → Open, then click "Open" in the dialog. This only needs to be done once. Alternatively: xattr -dr com.apple.quarantine linus-ai-macos-arm64
Linux — bash
$ chmod +x linus-ai-linux-x86_64
$ ./linus-ai-linux-x86_64 --version
LINUS-AI v1.4.0 (linux-x86_64)
 
# Install system-wide:
$ sudo mv linus-ai-linux-x86_64 /usr/local/bin/linus-ai
$ sudo chmod +x /usr/local/bin/linus-ai
 
# Verify and start:
$ linus-ai --activate LNAI-XXXX-XXXX-XXXX-XXXX
$ linus-ai --serve # Start API server on :8080
Windows — PowerShell
# Double-click the .exe to run, OR use PowerShell:
PS> .\linus-ai-windows-x86_64.exe --version
LINUS-AI v1.4.0 (windows-x86_64)
 
# Add to PATH (run as Administrator):
PS> Move-Item .\linus-ai-windows-x86_64.exe C:\Windows\System32\linus-ai.exe
 
# Activate and start:
PS> linus-ai --activate LNAI-XXXX-XXXX-XXXX-XXXX
PS> linus-ai --chat "Hello!"
Windows Defender: SmartScreen may flag the binary on first run. Click "More info" → "Run anyway". The binary is unsigned; checksums are provided above for verification.
Security

Verify Your Download

Always verify the SHA256 checksum of downloaded binaries before executing them.

Checksum verification
# Linux / macOS:
$ sha256sum linus-ai-linux-x86_64
d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3d4e5 linus-ai-linux-x86_64

$ shasum -a 256 linus-ai-macos-arm64
 
# Windows PowerShell:
PS> Get-FileHash .\linus-ai-windows-x86_64.exe -Algorithm SHA256 | Select Hash
Docker

Docker Image

Run LINUS-AI in an isolated container. The official image is under 60 MB and includes the binary and default model configs.

Docker
# Pull the latest image:
$ docker pull linusai/linus-ai:1.4.0
 
# Run the API server:
$ docker run -d \
--name linus-ai \
-p 8080:8080 \
-v ~/.linus_ai:/root/.linus_ai \
-e LINUS_AI_LICENSE=LNAI-XXXX-XXXX-XXXX-XXXX \
linusai/linus-ai:1.4.0 --serve
 
# GPU passthrough (NVIDIA):
$ docker run --gpus all -p 8080:8080 linusai/linus-ai:1.4.0 --serve --gpu
The Docker image does not include model weights. Mount your model directory with -v /path/to/models:/root/.linus_ai/models or use linus-ai --pull-model llama3.2 on first run.

Source

Build from Source

LINUS-AI is open-source for the core engine. Build from source for custom patches, unsupported platforms, or development.

⚠ Compile on Target — Pre-built binaries are provided only for the last tested release (v1.4.0). The main branch is development-in-progress and must be compiled directly on the target platform from the matching release tag. Cross-compilation is not guaranteed to produce a working binary. Always check out a release tag (e.g. git checkout v1.4.0) for stable builds.
Build instructions — macOS / Linux (click to expand) ▼
build on target — macOS / Linux
# Prerequisites: Rust 1.78+, Python 3.10+, cargo, cmake
$ git clone https://github.com/LINUS-AI-PRO/linus-ai.git
$ cd linus-ai
# Check out last tested release — do NOT build from main for production
$ git checkout v1.4.0
$ pip install -r requirements.txt
# macOS Apple Silicon (arm64)
$ cargo build --release --target aarch64-apple-darwin
# macOS Intel (x86_64)
$ cargo build --release --target x86_64-apple-darwin
# Linux x86_64
$ cargo build --release --target x86_64-unknown-linux-gnu
# Linux ARM64 (Graviton / Raspberry Pi 4+)
$ cargo build --release --target aarch64-unknown-linux-gnu
 
✓ Binary output: dist/linus-ai (symlinked from target/<triple>/release/linus-ai)
Build instructions — Windows (click to expand) ▼
build on target — Windows x86_64
# Prerequisites: Rust 1.78+, Python 3.10+, MSVC Build Tools 2022, cmake
PS> git clone https://github.com/LINUS-AI-PRO/linus-ai.git; cd linus-ai
PS> git checkout v1.4.0
PS> pip install -r requirements.txt
PS> cargo build --release --target x86_64-pc-windows-msvc
✓ Binary output: dist\linus-ai.exe (from target\x86_64-pc-windows-msvc\release\linus-ai.exe)

See the full BUILD.md on GitHub for GPU-specific flags (CUDA, Metal, ROCm), cross-compilation notes, and CI configuration.

v1.4.0

Release Highlights

What's new in LINUS-AI 1.4.0 — released March 2025.

⚡ Tensor Parallelism Across Multiple GPUs New

Split large models across up to 8 GPUs with automatic tensor sharding. 70B parameter models now run in real-time on 4× RTX 3090s.

🕸 Mesh Networking Mode New

Distribute inference across multiple machines on your private network. Nodes auto-discover each other using mDNS. No Kubernetes required.

🔐 Vault — Encrypted Memory Layer New

All conversation history and embeddings are stored AES-256-GCM encrypted at rest. The vault key never leaves your machine.

📦 Single Binary Consolidation Improved

All inference backends (CPU, CUDA, Metal, ROCm) are now compiled into a single binary. No more separate GPU builds — the binary auto-detects and uses the best available backend.

🦙 Llama 3.3, Mistral 3, Phi-4 Support Improved

Native support for the latest open-weight models with optimized GGUF quantization profiles (Q2_K through Q8_0). Custom model loading via TOML config.

🐛 Bug Fixes & Stability

Fixed Windows named pipe hang on long responses. Resolved macOS Metal memory leak on M3 Ultra. Linux NUMA node pinning now correctly isolates model threads. 23 additional issues resolved.
