Section 1

Installation

LINUS-AI ships as a single self-contained binary with no runtime dependencies. Download the appropriate build for your platform, mark it executable, and place it on your PATH. That's all there is to it.

System Requirements

RAM — 7B models
4 GB
Q4_K_M quantization
RAM — 13B models
16 GB
Q4_K_M quantization
RAM — 32B+ models
32 GB+
GPU VRAM or unified memory
Disk
5 GB+
Binary ~25 MB, models extra
OS
macOS 13+
Linux kernel 5.4+, Win 10 64-bit
GPU (optional)
CUDA 12 / ROCm / Metal
Significantly improves speed

macOS (Apple Silicon & Intel)

Choose the Apple Silicon build (darwin-arm64) for M1/M2/M3/M4 Macs. For older Intel Macs, use darwin-x86_64. The Metal backend is automatically used on Apple Silicon for GPU-accelerated inference.

macOS — Apple Silicon (M-series)
# Download the Apple Silicon binary
$ curl -Lo linus-ai https://github.com/LINUS-AI-PRO/linus-ai/releases/latest/download/linus-ai-darwin-arm64
 
# Make it executable and install system-wide
$ chmod +x linus-ai
$ sudo mv linus-ai /usr/local/bin/
 
# Verify installation
$ linus-ai --version
linus-ai 1.4.2 (darwin/arm64 · metal)
 
# Optional: shell completion (zsh)
$ linus-ai --completion zsh > ~/.zsh/completions/_linus-ai
$ echo 'fpath=(~/.zsh/completions $fpath)' >> ~/.zshrc && source ~/.zshrc
macOS — Intel (x86_64)
$ curl -Lo linus-ai https://github.com/LINUS-AI-PRO/linus-ai/releases/latest/download/linus-ai-darwin-x86_64
$ chmod +x linus-ai && sudo mv linus-ai /usr/local/bin/
🍺
Homebrew tap coming soon. A Homebrew tap (brew install linus-ai/tap/linus-ai) is planned for an upcoming release. Watch the GitHub repository for announcements.

Linux (x86_64 & ARM64)

LINUS-AI runs on any modern Linux distribution. CUDA-enabled builds include GPU support out of the box. After install, consider setting up a systemd service for automatic startup.

Linux x86_64
$ curl -Lo linus-ai https://github.com/LINUS-AI-PRO/linus-ai/releases/latest/download/linus-ai-linux-x86_64
$ chmod +x linus-ai && sudo mv linus-ai /usr/local/bin/
$ linus-ai --version
linus-ai 1.4.2 (linux/amd64 · cuda)
Linux ARM64 (Raspberry Pi 5, AWS Graviton, etc.)
$ curl -Lo linus-ai https://github.com/LINUS-AI-PRO/linus-ai/releases/latest/download/linus-ai-linux-arm64
$ chmod +x linus-ai && sudo mv linus-ai /usr/local/bin/

systemd Service (Linux)

To run LINUS-AI automatically on boot as a background server, create a systemd unit file. Replace your-user with your actual Linux username.

/etc/systemd/system/linus-ai.service
[Unit]
Description=LINUS-AI Private AI Inference Server
After=network.target
 
[Service]
Type=simple
User=your-user
ExecStart=/usr/local/bin/linus-ai --serve
Restart=on-failure
RestartSec=5
Environment=LINUS_AI_LICENSE_KEY=LNAI-XXXX-XXXX-XXXX-XXXX
 
[Install]
WantedBy=multi-user.target
 
# Enable and start
$ sudo systemctl daemon-reload
$ sudo systemctl enable --now linus-ai
$ sudo systemctl status linus-ai
● linus-ai.service — LINUS-AI Private AI Inference Server
Active: active (running) since …

Windows (PowerShell)

Download the Windows binary via PowerShell and add it to your PATH. Windows Terminal is recommended for the best experience. CUDA acceleration is supported on Windows with NVIDIA GPU drivers installed.

PowerShell (run as Administrator)
# Download the binary
PS> Invoke-WebRequest -Uri "https://github.com/LINUS-AI-PRO/linus-ai/releases/latest/download/linus-ai-windows-x86_64.exe" -OutFile "$env:LOCALAPPDATA\linus-ai\linus-ai.exe"
 
# Add to PATH permanently
PS> $path = [Environment]::GetEnvironmentVariable("Path","User")
PS> [Environment]::SetEnvironmentVariable("Path","$path;$env:LOCALAPPDATA\linus-ai","User")
 
# Restart terminal, then verify
PS> linus-ai --version
linus-ai 1.4.2 (windows/amd64 · cuda)
💡
WSL2 note. LINUS-AI works natively in WSL2 (Windows Subsystem for Linux). Use the Linux x86_64 binary inside your WSL2 environment. GPU passthrough to WSL2 via NVIDIA CUDA and AMD ROCm is supported — install the appropriate Windows GPU driver and WSL2 GPU package first.
Section 2

License Activation

LINUS-AI includes a 30-day free trial. After that, a license is required. Licenses are issued in the format LNAI-XXXX-XXXX-XXXX-XXXX and delivered to the email address used at checkout.

30-Day Trial

No action is required to start the trial. On first launch, LINUS-AI automatically enters trial mode. A banner is displayed at startup showing the remaining trial days. All features available on the Community plan are accessible during the trial.

Trial mode banner (startup)
╔══════════════════════════════════════════════════════╗
║ LINUS-AI • Trial Mode • 23 days remaining ║
║ Activate: linus-ai --activate LNAI-XXXX-XXXX-XXXX-XXXX ║
╚══════════════════════════════════════════════════════╝

CLI Activation

Activate with your license key and the email address used at purchase. An internet connection is required only for the initial activation handshake. After that, the binary operates fully offline.

License activation
$ linus-ai --activate LNAI-XXXX-XXXX-XXXX-XXXX --activate-email user@company.com
Contacting license server…
✓ License activated — Professional · seat 1/5
✓ Machine fingerprint recorded
✓ License saved to ~/.linus_ai/config.toml

Config File Activation

You can also add the license key directly to the configuration file. This is convenient for automated deployments or when managing multiple machines.

~/.linus_ai/config.toml
[license]
key = "LNAI-XXXX-XXXX-XXXX-XXXX"
email = "user@company.com"

Environment Variable

For CI/CD pipelines or containerised deployments, supply the key via an environment variable. The email is not required when using the environment variable method after the initial machine binding has been completed.

Environment variable
$ export LINUS_AI_LICENSE_KEY="LNAI-XXXX-XXXX-XXXX-XXXX"
$ linus-ai --serve
✓ License valid (Professional)

Check License Status

License status
$ linus-ai --license-status
Plan : Professional
Email : user@company.com
Seats : 1 / 5
Expires : perpetual
Machine : bound (fingerprint: 9a2f…c41b)
Last check : 2026-03-15

Trial Expiry

When the 30-day trial expires, LINUS-AI enters read-only mode. Existing conversations in the vault remain accessible, but new inference requests are blocked until a valid license key is activated. The control panel GUI continues to load and will display an activation prompt.

Machine Binding (Perpetual Licenses)

Perpetual licenses are bound to the machine on which they are first activated. The binding uses a hardware fingerprint derived from CPU serial, motherboard UUID, and primary network interface MAC address. Virtual machines generate stable fingerprints. Each seat in your license can be bound to one machine at a time.

Deactivate & Move to Another Machine

To move a license seat to a new machine, deactivate it on the current machine first. This releases the machine binding and allows activation on a different device.

Deactivate license
$ linus-ai --deactivate
Deactivating machine binding…
✓ Seat released — you can now activate on another machine
⚠
Lost machine access. If a machine is lost or destroyed before deactivating, contact support@linus-ai.com with your license key reference to request a manual seat release. Enterprise plans include self-serve seat management in the admin portal.
Section 3

Model Management

LINUS-AI uses the Ollama model registry for downloading and managing models. Models are stored locally in ~/.linus_ai/models/ and are never uploaded or shared. Once downloaded, models work fully offline.

Pulling, Listing, and Deleting Models

Model management commands
# Pull (download) a model from the registry
$ linus-ai --pull-model llama3.2
Pulling llama3.2 (Q4_K_M · 2.0 GB)…
████████████████████████████████ 100% · 42 MB/s
✓ llama3.2 ready — ~/.linus_ai/models/llama3.2
 
# Pull with a specific quantization level
$ linus-ai --pull-model llama3.2:Q8_0
 
# List locally installed models
$ linus-ai --list-models
NAME SIZE QUANT MODIFIED
llama3.2 2.0 GB Q4_K_M 2026-03-14
mistral:7b 4.1 GB Q4_K_M 2026-03-10
qwen2.5:7b 4.7 GB Q4_K_M 2026-03-08
 
# Delete a model
$ linus-ai --delete-model llama3.2
✓ llama3.2 deleted

Quantization Levels

Quantization reduces model size and memory usage at a small cost to output quality. Q4_K_M is the recommended default for most users — it offers an excellent balance of speed, memory, and quality.

Level Size (7B model) VRAM (7B) Speed Quality Best for
Q2_K ~1.6 GB ~2 GB Fastest Lowest Memory-constrained devices, quick prototyping
Q3_K_M ~2.1 GB ~3 GB Very fast Fair 4 GB RAM systems, edge inference
Q4_K_M ~2.8 GB ~4 GB Fast Good Recommended default for most users
Q5_K_M ~3.4 GB ~5 GB Good Very good Higher quality without full precision overhead
Q8_0 ~6.7 GB ~8 GB Moderate Excellent Near-lossless inference, 8 GB VRAM GPUs
F16 ~13.5 GB ~16 GB Slowest Best Reference quality, fine-tuning, benchmarking

Section 4

Inference Modes

LINUS-AI automatically selects the best inference backend for your hardware. You can override this with the backend config key or the --backend CLI flag.

Backend Flag Hardware Notes
auto --backend auto Any Recommended. Detects Metal → CUDA → ROCm → CPU in order.
cpu --backend cpu Any CPU Force CPU-only. Works everywhere, slowest for large models.
cuda --backend cuda NVIDIA GPU Requires CUDA 12+ drivers. Fastest on NVIDIA hardware.
metal --backend metal Apple Silicon Uses Apple GPU via Metal. Unified memory — no VRAM limit.
rocm --backend rocm AMD GPU Requires ROCm 5.7+ (Linux). RX 6000/7000 series recommended.

Backend Detection Output

Startup — backend auto-detection
$ linus-ai --serve --verbose
Detecting inference backend…
[✓] Metal Apple M3 Pro · 18 GPU cores · 36 GB unified
[✗] CUDA No NVIDIA GPU detected
[✗] ROCm No AMD GPU detected
[✓] CPU 12-core arm64 (fallback)
Selected: metal
Loading llama3.2 (Q4_K_M) → Metal … done (1.2 s)
✓ API server: http://127.0.0.1:8080

GPU Layers

The gpu_layers setting controls how many model layers are offloaded to the GPU. -1 offloads all possible layers (maximum GPU acceleration). Reduce this number if you hit out-of-memory errors.

~/.linus_ai/config.toml — GPU layers
[inference]
gpu_layers = -1 # -1 = offload all layers (default)
gpu_layers = 32 # offload first 32 layers only
gpu_layers = 0 # CPU-only (same as backend = "cpu")

CPU Threads

CPU thread count is automatically detected from your system's logical core count. Override it with threads = N in config or --threads N on the command line. For best performance, set threads to the number of physical (not logical) cores.

CPU thread override
[inference]
threads = 8 # Override auto-detected thread count

Context Length

Context length (in tokens) determines how much conversation history and document content can be kept in a single inference pass. Larger contexts use more memory. Supported models (e.g., Qwen2.5, Llama 3.1) support up to 128K tokens.

Context length configuration
[inference]
context_len = 8192 # default: 8192 tokens
context_len = 32768 # 32K — for longer documents
context_len = 131072 # 128K — supported models only

Section 6

Custom System Prompts

Every profile ships with a carefully crafted default system prompt. You can override it at any level — globally via config, per-request via the API, or interactively in the GUI — without affecting the profile selection for other sessions.

Override via GUI

Open the control panel at http://localhost:8080, navigate to the Setup tab, and find the System Prompt Customization card. Edit the text area and click Save. The override is stored per-profile in your browser's localStorage, so each profile remembers its own custom prompt independently. Click Reset to Default at any time to restore the profile's built-in prompt.

Override via API

Pass a system field in the request body when calling POST /agent/stream or the OpenAI-compatible POST /v1/chat/completions endpoint.

API — custom system prompt
$ curl http://localhost:8080/v1/chat/completions \
-H "Content-Type: application/json" \
-d '{
"model": "llama3.2",
"system": "You are a pirate assistant. Respond only in pirate speak.",
"messages": [{"role":"user","content":"Tell me about the weather."}]
}'

Override via Config File

Set a global system prompt in ~/.linus_ai/config.toml. This applies to all CLI and API requests that do not supply their own system field. Profile-level prompts take precedence over this global setting unless you explicitly set the profile to none.

~/.linus_ai/config.toml
[inference]
system_prompt = "You are an internal assistant for Acme Corp. Only answer questions related to our products and policies."
💾
Per-profile localStorage. Custom prompts entered in the GUI Setup tab are saved in your browser's localStorage under a key per profile (e.g., lnai_prompt_code). Clearing browser data will reset them. For permanent overrides, use the config file method instead.

Control Panel GUI

LINUS-AI includes a built-in web control panel. Start it with linus-ai --serve and open http://localhost:8080 in any modern browser. No additional installation or configuration is needed.

Start the control panel

Agent Tab

Multi-turn agentic mode where the model can use tools to complete complex tasks. The agent can browse the web, read files, execute code, and call external APIs — all locally, all private.

  • Tool-use loop — model autonomously plans, acts, observes, and iterates
  • Built-in tools: file reader, web search (local), code execution sandbox
  • Step-by-step trace — view every reasoning step and tool call
  • Interrupt and redirect — stop mid-task and give new instructions
  • Available on Professional, Team, and Enterprise plans

Section 8

CLI Reference

All flags can also be set via ~/.linus_ai/config.toml or environment variables. CLI flags take highest precedence, followed by environment variables, then the config file.

Flag Type / Default Description
Server
--serve flag Start the HTTP server and control panel GUI.
--host HOST string · 127.0.0.1 Bind address. Use 0.0.0.0 to expose on the network.
--port PORT int · 8080 HTTP port for the server and control panel.
Chat & Inference
--chat MESSAGE string Send a one-shot message and print the response, then exit.
--model NAME string · config default Model to load (e.g. llama3.2, mistral:7b).
--profile NAME string · general AI vertical profile. See the Profiles section for all 14 IDs.
--backend MODE string · auto Inference backend: auto | cpu | cuda | metal | rocm.
--threads N int · auto CPU threads for inference. Defaults to physical core count.
Model Management
--pull-model NAME string Download a model from the Ollama registry (e.g. llama3.2, qwen2.5:32b).
--list-models flag List all locally installed models with size, quantization, and date.
--delete-model NAME string Delete a locally installed model and free disk space.
License
--activate KEY string Activate a license key (LNAI-XXXX-XXXX-XXXX-XXXX format).
--activate-email EMAIL string Email address associated with the license (required with --activate).
--license-status flag Display current license status, plan, seats, and machine binding.
--deactivate flag Release the machine binding for this seat so it can be used elsewhere.
Distributed / Mesh
--tensor-parallel N int · 1 Split model across N GPUs using tensor parallelism.
--mesh-role ROLE string Mesh role: coordinator or worker.
--mesh-join HOST:PORT string Join an existing mesh cluster (worker nodes use this to connect to the coordinator).
General
--config PATH string · ~/.linus_ai/config.toml Path to a custom configuration file.
--quiet flag Suppress all non-essential output. Useful in scripts.
--verbose flag Enable detailed debug logging including backend detection and token stats.
--version flag Print version, build platform, and active backend, then exit.
Section 9

Troubleshooting

Common issues and their solutions. If you cannot find a fix here, check the log file at ~/.linus_ai/logs/linus-ai.log and include the relevant excerpts when reporting an issue.

Model not found / "no such model: llama3.2"

This error means the model has not been downloaded yet. Pull it from the registry:

$ linus-ai --pull-model llama3.2
$ linus-ai --list-models

If you already have the model, check that the name matches exactly (case-sensitive, including any tag suffix such as :7b or :Q4_K_M). Also verify that ~/.linus_ai/models/ is on the same filesystem as expected — symbolic links to different volumes are supported.

Out of memory / "CUDA out of memory" / model crashes

The model requires more GPU VRAM than available. Try one or more of the following:

  • Reduce gpu_layers to offload fewer layers (e.g., gpu_layers = 20)
  • Use a lower quantization level: switch from Q8_0 to Q4_K_M
  • Reduce context_len — long contexts use significant KV cache memory
  • Close other GPU workloads (browsers with WebGPU, other ML frameworks)
  • Switch to a smaller model (e.g., 7B instead of 13B)
Reduce GPU offload
[inference]
gpu_layers = 20 # reduced from -1
context_len = 4096 # reduced from 8192
quantize = "Q4_K_M" # if using Q8_0, switch to this
License not recognized / "invalid license key format"

Check the following:

  • The key must be in the exact format LNAI-XXXX-XXXX-XXXX-XXXX — four groups of four characters separated by hyphens, prefixed with LNAI-
  • The email address must exactly match the one used at checkout (case-insensitive)
  • Check for accidental leading/trailing whitespace if pasting from email
  • If you receive "key already activated on another machine", run linus-ai --deactivate on the old machine first, or contact support
$ linus-ai --activate "LNAI-ABCD-EFGH-IJKL-MNOP" --activate-email "you@example.com"
$ linus-ai --license-status
Port 8080 already in use

Another process is using port 8080. Start LINUS-AI on a different port:

$ linus-ai --serve --port 8081
✓ Control panel: http://127.0.0.1:8081

Or set it permanently in config: [server] port = 8081. To find the conflicting process on macOS/Linux: lsof -i :8080.

Metal backend crashes on Apple Silicon

Metal-related crashes are typically caused by one of:

  • macOS version too old — update to macOS 14 (Sonoma) or later. Metal shader features used by LINUS-AI require macOS 14+.
  • Xcode Command Line Tools not installed — run xcode-select --install in Terminal.
  • Metal driver regression — if a recent macOS update broke Metal, add backend = "cpu" to your config temporarily as a workaround.
$ xcode-select --install
$ sw_vers
ProductVersion: 14.7.4

Log Location

Detailed logs are written to ~/.linus_ai/logs/linus-ai.log. The log rotates daily and keeps 7 days of history. Pass --verbose at startup to increase log verbosity for debugging. When reporting an issue on GitHub, please include the last 50 lines of the log.

View the log
$ tail -f ~/.linus_ai/logs/linus-ai.log
[2026-03-15 09:12:44] INFO Server started on 127.0.0.1:8080
[2026-03-15 09:12:44] INFO Backend: metal (Apple M3 Pro)
[2026-03-15 09:12:45] INFO Model loaded: llama3.2 Q4_K_M (1241ms)
